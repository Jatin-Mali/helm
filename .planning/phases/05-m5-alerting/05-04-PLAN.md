<task type="auto">
  <name>T5.4.2: Implement AlertSink for PagerDutySink with lifecycle-aware actions</name>
  <files>crates/helm-monitor/src/alerting/pagerduty.rs</files>
  <action>
Implement `impl AlertSink for PagerDutySink`:

```
async fn send(&self, alert: &AlertPayload) -> Result<(), Box<dyn std::error::Error>>
```

Algorithm:
1. Determine event_action based on alert.lifecycle:
   - If lifecycle == Open: event_action = "trigger"
   - If lifecycle == Resolved: event_action = "resolve"
2. Map alert.severity to PD severity string:
   - Severity::Critical → "critical"
   - Severity::Warning → "warning"
   - Severity::Info → "info"
3. Build PdEvent:
   - routing_key: self.routing_key.clone()
   - event_action: from step 1
   - dedup_key: alert.fingerprint.clone()
   - payload.summary: format!("{}: {}", alert.severity.as_str().to_uppercase(), alert.title)
   - payload.severity: from step 2
   - payload.source: alert.detector_id.clone()
   - payload.custom_details: json!({ "fingerprint": alert.fingerprint, "affected_resource": alert.affected_resource, "description": alert.description, "timestamp": alert.timestamp })
4. Serialize PdEvent to JSON
5. POST to https://events.pagerduty.com/v2/enqueue with Content-Type: application/json
6. If response is 202 Accepted: return Ok(())
7. If response is 4xx: return Err (no retry)
8. If response is 5xx or network error: return Err (let AlertRouter handle retry at its level if needed)

Log the event_action, dedup_key, and response status at INFO/WARN level.
