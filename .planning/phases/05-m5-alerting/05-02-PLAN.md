<task type="auto">
  <name>T5.2.2: Implement AlertSink for WebhookSink with exponential backoff</name>
  <files>crates/helm-monitor/src/alerting/webhook.rs</files>
  <action>
Implement `impl AlertSink for WebhookSink`:

```
async fn send(&self, alert: &AlertPayload) -> Result<(), Box<dyn std::error::Error>>
```

Algorithm:
1. Serialize alert to JSON: `serde_json::to_string(&alert)?`
2. Retry loop: for attempt in 0..3:
   a. POST to self.url with body=json, Content-Type=application/json
   b. If response.status().is_server_error() || network error: continue to next attempt with delay
   c. If response.status().is_client_error(): return Err immediately (no retry)
   d. If response.status().is_success(): return Ok(())
   e. Between attempts: sleep for 2^attempt seconds (1s, 2s, 4s)
3. After 3 failed attempts: return Err with message "Webhook delivery failed after 3 attempts"

Use tokio::time::sleep() for delays. Log each attempt at WARN level.
