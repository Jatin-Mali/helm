  <name>T5.3.2: Implement AlertSink for SlackSink with message formatting</name>
  <files>crates/helm-monitor/src/alerting/slack.rs</files>
  <action>
Implement `impl AlertSink for SlackSink`:

```
async fn send(&self, alert: &AlertPayload) -> Result<(), Box<dyn std::error::Error>>
```

Algorithm:
1. Build Slack message JSON (simple format or Block Kit; simple format shown):
   ```json
   {
     "channel": "self.channel.as_deref().unwrap_or(...)",
     "attachments": [
       {
         "fallback": "{severity}: {title}",
         "color": severity_to_color(alert.severity),
         "title": alert.title,
         "fields": [
           { "title": "Severity", "value": alert.severity.as_str(), "short": true },
           { "title": "Detector", "value": alert.detector_id, "short": true },
           { "title": "Resource", "value": alert.affected_resource, "short": false },
           { "title": "Fingerprint", "value": alert.fingerprint, "short": false },
           { "title": "Category", "value": alert.category.as_str(), "short": true },
           { "title": "Timestamp", "value": "{timestamp iso8601}", "short": true }
         ]
       }
     ]
   }
   ```
   
2. Serialize to JSON
3. POST to self.webhook_url with Content-Type: application/json
4. If response is 200 OK: return Ok(())
5. If response is 4xx/5xx: return Err with status

Log the POST attempt at DEBUG level; log errors at WARN level.
