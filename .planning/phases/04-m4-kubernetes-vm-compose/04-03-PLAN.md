- Primary command: `docker ps -a --filter "label=com.docker.compose.project" --format "{{.Names}}\t{{.Label \"com.docker.compose.project\"}}\t..."`
- Grouping strategy: Parse container list, group by project label, calculate per-project aggregates (container_count, health status)
- Graceful degradation: if docker missing, return default with available=false

<task type="auto">
  <name>Task 3: Implement docker-compose project parser and grouping logic</name>
  <files>crates/helm-monitor/src/collectors/compose.rs</files>
  <action>
Add to compose.rs:
- `fn parse_compose_projects(s: &str) -> Vec<ComposeProject>`
  - Parse JSON output (or CSV if JSON unavailable) from docker ps
  - Group containers by com.docker.compose.project label
  - For each project:
    - Extract name from label
    - Count total containers
    - Count running containers (filter by State==running)
    - Count healthy containers (filter by Health==healthy or health status == "healthy")
    - Collect container names into Vec<String>
    - Derive status: "healthy" if all running and all healthy, "degraded" if some running, "down" if none running
  - Return Vec<ComposeProject> sorted by name
- Use standard JSON serde parsing if possible; fallback to manual text parsing if JSON unavailable
- Use `err()` helper for parse failures
