<context>
**Detector trait:** `crates/helm-monitor/src/detectors/mod.rs` lines 25–35 (actual signature):
```rust
pub trait Detector: Send + Sync {
    fn id(&self) -> &'static str;
    fn domain(&self) -> &'static str;
    fn detect(&self, snapshot: &SystemSnapshot, previous: Option<&SystemSnapshot>) -> Vec<Finding>;
}
```
**Note:** NOT async. Takes `&SystemSnapshot` (not `SnapshotDomains`). Returns `Vec<Finding>` (not Result). Includes optional `previous` snapshot. Access kubernetes data via `snapshot.domains.kubernetes`, libvirt via `snapshot.domains.libvirt`, compose via `snapshot.domains.compose`.

**Registry pattern:** `mod.rs` lines 77–98 shows how to register detectors in `default_registry()`.

**Analog detector:** `crates/helm-monitor/src/detectors/container_restart.rs` shows:
- Struct + impl Detector
- Read from snapshot field (containers)
- Loop and check threshold
- Generate Finding with fingerprint, severity, message
- Use findings helper functions

**Finding struct:** `findings.rs` includes:
- `pub struct Finding { pub fingerprint: String, pub severity: Severity, pub domain: String, pub title: String, pub detail: String, pub evidence: String }`

**Severity levels:** CRIT, WARN, INFO (enum in findings.rs)

**Threshold config:** Lookup pattern in detector code: `profile.get_threshold("kubernetes.pod_restart_max", 5)` or similar.
</context>
